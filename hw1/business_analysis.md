# Бизнес-анализ проекта «iNat Migrate Vietnam»

Делаю аналитический сервис на открытых данных iNaturalist - про распределение, миграции и фенологию видов во Вьетнаме. Данные тяну из iNaturalist Open Dataset (`s3://inaturalist-open-data/`), он публичный и бесплатный.

## Организационная структура

**Заказчик (условный):** исследовательская группа по биоразнообразию. За прототип взял Институт экологии и биологических ресурсов Вьетнама (IEBR, Ханой) - реальный профильный институт. Подходит и любая университетская лаборатория, которая занимается фауной Юго-Восточной Азии.

**Кто будет пользоваться:**

- ежедневно - полевые исследователи, аспиранты-биологи, координаторы citizen-science программ;
- эпизодически - природоохранные NGO (WWF Vietnam, Wildlife Conservation Society) и госорганы по охраняемым территориям; они потребляют отчёты и карты, а не сидят в системе каждый день;
- сообщество iNaturalist (десятки тысяч наблюдателей) - не пользователи сервиса в классическом смысле, а поставщики данных, на которых он работает.

**Команда:** один человек - я. Совмещаю роли PM, дата-инженера, ML-исследователя и разработчика. Помимо себя - эксперт-биолог из университета на консультации.

## Контакты и рабочие чаты

Полноценные корпоративные коммуникации мне не нужны. Использую:

- **Локальные заметки** - журнал решений и экспериментов.

## Бизнес-цель проекта

Дать исследователям биоразнообразия Вьетнама автоматический аналитический инструмент над открытыми данными iNaturalist - чтобы вместо ручного парсинга CSV они получали готовые научные артефакты:

1. определение вида по фотографии (CV-модель);
2. предсказание вероятности встречи вида в координате и дате `(lat, lon, date) → P(species)`. Это задача SDM (Species Distribution Modeling, моделирование пространственного распределения видов), у нас с дополнительной временной осью;
3. визуализация миграционных коридоров, фенологических календарей и сдвигов ареалов;
4. автоматическое выявление нетипичных встреч (потенциальные locality records, инвазии, ошибки определения);
5. инвентаризация эндемиков и регионов с дефицитом наблюдений (white spots);
6. карточка вида (Species profile) - по фотографии получить название, галерею других фото этого вида, карту ареала, фенологический календарь и список похожих видов; главный пользовательский экран сервиса.

**Целевые показатели первой итерации:**

- CV-модель: top-1 ≥ 65%, top-5 ≥ 85%, macro-F1 ≥ 0.55 на тест-сете.
- SDM-модель: mean ROC-AUC ≥ 0.80, mAP ≥ 0.35 по видам с ≥30 наблюдениями.
- Аналитика признаётся полезной экспертом-биологом (precision на anomaly detector ≥ 70%).

**Бизнес-модель (на будущее, вне MVP):** freemium по образцу GBIF. Базовые карты и определение видов - бесплатно. Подписки для исследовательских организаций - bulk-export, кастомные SDM под их выборку, alerts на аномалии, автогенерация отчётов.

## Существующие решения и их недостатки

### iNaturalist

- **Что делает:** citizen-science платформа - загрузка наблюдений, автоопределение видов, верификация сообществом.
- **Почему недостаточно:** ориентирована на сбор и определение, а не на исследовательскую аналитику. Нет встроенных SDM, миграционных коридоров, фенологии, anomaly detection. Координаты уязвимых видов загрублены до ячейки ~25×25 км - для миграционного анализа эти 12% наблюдений неинформативны. CV-модель глобальная, без региональной доводки под Вьетнам.

### eBird Status & Trends (Cornell Lab)

- **Что делает:** аналитика миграций птиц на основе миллиардов наблюдений.
- **Почему недостаточно:** только птицы. Доступ к Basic Dataset через заявку, не open. Глобальная модель без региональной специализации.

### Merlin Bird ID, BirdNET (Cornell Lab)

- **Что делает:** распознавание птиц по фото и звуку для любителей.
- **Почему недостаточно:** закрытые модели, не для исследователей; нет API для аналитики; не работают за пределами орнитологии.

### Pl@ntNet

- **Что делает:** распознавание растений.
- **Почему недостаточно:** только растения; нет пространственно-временной аналитики.

### GBIF

- **Что делает:** агрегатор биоразнообразия, включая зеркало iNat.
- **Почему недостаточно:** сырые occurrence-данные в формате Darwin Core; нет встроенного распознавания, нет UX для аналитика, нет SDM «из коробки».

**Чем мой сервис отличается.** Это аналитическая надстройка над citizen-science данными iNaturalist для пока что одной страны (Вьетнам). Что внутри:

- региональная CV-модель, обученная только на видах Вьетнама - на эндемиках точность выше, чем у глобальной модели iNat;
- geo-temporal SDM `(lat, lon, date) → P(species)` для всех ~8 000 видов research-grade;
- аналитика поверх SDM: anomaly detection, reverse query, phenocorrelation, endemic explorer, migration corridors, white spots map;
- карточка вида (Species profile) - единый пользовательский экран, в котором собраны выходы всех компонентов;
- честная обработка obscured-наблюдений: отделяю от обучающей выборки, отдельно помечаю в визуализации;
- открытый код и воспроизводимый pipeline - DVC-снапшот iNat-дампа на момент обучения.

**С iNaturalist по сбору данных не конкурирую.** Беру их Open Data dump (`s3://inaturalist-open-data/`, обновляется раз в месяц, доступ публичный). То же самое делают GBIF и eBird Status & Trends - агрегация и аналитика, а не свой сбор.

## 1.1 Текущая ситуация (Assessing current solution)

### Доступные ресурсы

**Железо.** Локальная машина:

- GPU: NVIDIA RTX 4050 Laptop, **6 GB VRAM** - хватает для CV-моделей до ~25M параметров с mixed precision (fp16) и gradient accumulation; ResNet50, EfficientNet-B2 и ViT-Small помещаются.
- RAM: **32 GB** - observations.csv.gz и photos.csv.gz Вьетнама легко держатся в памяти целиком в pandas/duckdb без чанкования.
- Свободное место: **>100 GB** - хватает на метаданные, фото в `medium`-разрешении (~20 GB на ~129K фото), чекпоинты моделей и DVC-кэш.
- Закупка железа не требуется. Резерв на финальное обучение тяжёлых моделей - Google Colab (T4 16 GB free / Pro A100).

**Данные.** Источник - iNaturalist Open Dataset на AWS Open Data:

- Бакет `s3://inaturalist-open-data/`, публичный доступ без AWS-аккаунта.
- 6 gzip-CSV метаданных (`observations`, `photos`, `taxa`, `observers`, `projects`, `observations_projects`), несколько GB суммарно.
- ~129 000 research-grade наблюдений по Вьетнаму, ~129 000 с фото (99.8%), ~88% с открытыми координатами, 8 276 уникальных видов. По таксонам: птицы 43K, насекомые 33K, растения 18K, рептилии 14K, млекопитающие 6K, амфибии 5K (на 2026-04-29).
- Лицензии у каждого фото свои (поле `license`): CC0, CC-BY, CC-BY-NC. CC-BY-NC исключаю, чтобы не получить юридических ограничений; остаётся CC0 и CC-BY.
- Дополнительный канал - публичный API iNaturalist (`api.inaturalist.org/v1`) для разведки и обогащения, но не как основной массовый источник (лимиты).
- Закупать или собирать дополнительные данные не нужно. Всё открыто и доступно без переговоров с заказчиком.

**Эксперты.** Биолог из университета. Что от него нужно:

- консультации раз в 2 недели - спорные определения, валидация аномалий (50–100 наблюдений за сессию);
- сверка миграционных коридоров, которые выдаёт модель, с известной литературой по фауне Юго-Восточной Азии.

Работа с экспертом асинхронная - основной поток не блокирует.

### Риски и план снижения

**1. Срыв сроков (плотный график, 10 недель на 6 фаз).**

- Вероятность: средняя. Влияние: высокое.
- План снижения: жёсткая приоритизация задач внутри каждой фазы; еженедельный контроль прогресса по чек-листу артефактов; не вытягиваю фичи из roadmap в текущую итерацию.

**2. Малое количество фото у редких видов (long tail).**

- Вероятность: высокая. Влияние: среднее.
- План снижения: классифицирую только виды с ≥50 фото; для остальных - иерархическая метка до рода/семейства. SDM работает с любым количеством точек.

**3. Sampling bias данных iNat (города, выходные, харизматичные виды).**

- Вероятность: высокая. Влияние: среднее.
- План снижения: отделяю `geoprivacy=open` от `obscured`; фильтр `positional_accuracy ≤ 1000 м`; полная коррекция bias (target-group background) - в roadmap.

**4. Obscured-наблюдения для уязвимых видов (12% базы).**

- Вероятность: 100%. Влияние: низкое.
- План снижения: отделяю явно, не использую в обучении SDM; в аналитике помечаю как «приблизительная локация».

**5. Ограничения GPU 6 GB VRAM.**

- Вероятность: 100%. Влияние: среднее.
- План снижения: mixed precision (fp16); разрешение 224×224 как baseline; gradient accumulation для эффективного batch=64+; pretrained backbone и fine-tuning. Резерв - Colab.

**6. Дрейф iNat dump (ежемесячные обновления).**

- Вероятность: средняя. Влияние: низкое.
- План снижения: DVC-снапшот в начале фазы 2; не делаю ребилд во время курса.

**7. Слабые закономерности `(lat, lon, date) → species` для повсеместных видов.**

- Вероятность: средняя. Влияние: среднее.
- План снижения: честные baselines (random, by-prevalence, by-region); если SDM не бьёт baseline - смещаю фокус на CV + миграционную аналитику; в Evaluation отражаю.

**8. Лицензионные ограничения фото (CC-BY-NC).**

- Вероятность: низкая. Влияние: низкое.
- План снижения: фильтр по `license` на этапе отбора (только CC0 + CC-BY); атрибуция CC-BY в финальном отчёте.

**9. Зависимость от расписания эксперта-биолога.**

- Вероятность: низкая. Влияние: низкое.
- План снижения: пакетные не блокирующие консультации раз в 2 недели; основной поток автоматический.

## 1.2 Решаемые задачи с точки зрения аналитики (Data Mining goals)

В проекте две независимые ML-модели + аналитический слой поверх них. Для моделей применяю объективные метрики, для аналитики - экспертную и литературную валидацию.

### CV-классификатор: фото → species

**Метрики:**

- **Top-1 accuracy** - основная: вид угадан с первой попытки.
- **Top-5 accuracy** - попадание правильного вида в пятёрку наиболее вероятных. Стандарт fine-grained классификации.
- **Macro-F1** - среднее F1 по классам без учёта частоты, защищает от смещения к массовым видам.
- **Per-class precision/recall** - для разбора ошибок и выявления визуально похожих пар.

**Пороги успеха:**

| Уровень              | Top-1 | Top-5 | Macro-F1 |
| -------------------- | ----- | ----- | -------- |
| Минимально приемлемо | 50%   | 75%   | 0.40     |
| Целевой              | 65%   | 85%   | 0.55     |
| Отлично              | 75%+  | 92%+  | 0.65+    |

State of art на iNat-2021 (10 000 видов, миллионы фото) - 92% top-1 на ViT-Huge. На региональном датасете Вьетнама с ~1 000 видов после фильтра по ≥50 фото реалистично ожидать 60–75% top-1, что соответствует публикациям по региональным fine-grained датасетам.

### Geo-temporal SDM: `(lat, lon, date) → P(species)`

**Метрики:**

- **Mean ROC-AUC** (по видам, потом усреднение) - стандарт литературы SDM.
- **Mean Average Precision (mAP)** - устойчива к несбалансированности; важна для редких видов.
- **Brier score / log-loss** - для калибровки вероятностей; критично для anomaly detector, где нужна корректная оценка `P(species | location, date)`.

**Пороги успеха:**

| Уровень              | Mean ROC-AUC | mAP   |
| -------------------- | ------------ | ----- |
| Минимально приемлемо | 0.70         | 0.20  |
| Целевой              | 0.80         | 0.35  |
| Отлично              | 0.85+        | 0.50+ |

Mac Aodha et al. (ICCV 2019, GeoPriors) на глобальных iNat-данных получили mean ROC-AUC ~0.85 на семействах. На региональном Вьетнаме реалистично 0.75–0.82.

### Аналитический слой

Все остальные компоненты - anomaly detector, migration corridors, phenology, endemic explorer, white spots, reverse query, species profile - это аналитика поверх SDM и сырых данных. ML-метрики у них нет, проверяю по-другому:

- **Anomaly detector:** биолог просматривает 50 топ-аномалий, считаю precision. Целевой ≥70%, то есть из 10 алертов хотя бы 7 признаны интересными - реальные locality records, ошибки определения или инвазии.
- **Migration corridors:** сверяю с литературой. Для перелётных птиц на East Asian-Australasian Flyway известные коридоры должны на модели воспроизводиться, иначе модель сломана.
- **Phenology:** сверяю с фенологическими календарями для модельных видов.
- **Endemic explorer, white spots, reverse query:** usability-сессия с экспертом 30 минут - что нашёл, какие были косяки UX.
- **Species profile:** usability-сессия. Проверяю: научное и общеупотребительные названия берутся правильно (из `taxa.csv` iNat), в галерее минимум 8 фото когда они есть, SDM-карта и фенологический календарь отображаются, похожие виды реально похожи.

Всё это пишу в `hw5/evaluation.md` - что эксперт сказал, что вылезло при сверке.

### MVP первой итерации (9 фич + 2 модели)

**Базовые модели (2):**

1. **CV-классификатор:** фото → species. Primary - ViT-Small с DINOv2 pretraining; baselines - EfficientNet-B2 и ResNet50.
2. **Geo-temporal SDM:** `(lat, lon, date) → P(species)` - ядро проекта.

**Аналитика поверх моделей (7):**

3. **Anomaly detector** - флаг «нетипичная встреча» по порогу выхода SDM.
4. **Reverse query** - какие виды наиболее вероятны в (lat, lon) в дату D.
5. **Phenocorrelation** - связи видов по времени (lag-correlation), включая опылитель–растение.
6. **Endemic explorer** - карты и фенология эндемиков Вьетнама.
7. **Migration corridors** - анимированные карты «волн» по неделям года для мигрирующих видов.
8. **White spots map** - поверхность sampling effort, выделение недонаблюдённых регионов.
9. **Species profile (карточка вида)** - главный пользовательский экран: научное и общеупотребительные названия, галерея фото вида из iNat, карта ареала (наблюдения + SDM-предсказание), фенологический календарь, эндемизм, список похожих видов (по эмбеддингам CV-модели).

Девять фич + два ядра - это весь MVP. Фичи 3–9 не требуют своих моделей, они переиспользуют SDM и сырые данные iNat. Единственное исключение - мини-модель sampling effort под White spots. Species profile (фича 9) - главный экран приложения, в нём собираются выходы остальных компонентов в одну карточку.

### Roadmap (вне MVP)

- **Threatened species heatmap** - слой IUCN Red List.
- **Bias correction toggle** - target-group background, методическая коррекция sampling bias.
- **Climate-driven range projections** - WorldClim + CMIP6 climate scenarios, предсказание сдвигов ареалов при изменении климата.
- **Auto-reports** - PDF-отчёты по запросу (регион + период) с автогенерацией всех артефактов.
- **Платный API и подписки** для исследовательских организаций (bulk export, кастомные SDM, alerts).

### Ресурсы и зависимости

- **Время:** ~10–13 часов в неделю × 10 недель = 100–130 часов.
- **Деньги:** ноль. Open data, локальное железо, при нужде Colab бесплатно.
- **От чего завишу:** эксперт-биолог раз в 2 недели в фазах 4–5; работоспособность машины (RTX 4050) на протяжении 3-недельной фазы Modeling.
